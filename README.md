# 抖音福袋助手 · 用户使用指南

## 一、准备工作
1. 下载安装

## 二、常规操作流程
1. 点击页面右上角的 **“添加账号”**，为每个账号扫码登录。成功后对应账号会出现在“账号列表”中。
2. 每个账号点击“启动”后，系统会自动扫描、预热并参与福袋。
3. 观察左侧参数配置可开启/关闭：过滤粉丝团、只抢实物、优先大红包、自动抢红包、随机互动等。
4. 日志面板里会实时显示扫描、候选、参与和中奖情况；福袋面板展示当前参与的福袋信息。
5. 需要观察服务端状态时，切换到日志的“学习”标签或访问 `http://<服务端地址>/stats`。

## 三、参数说明（面板左侧）
| 项目 | 作用 |
| --- | --- |
| 过滤粉丝团 | 打开后跳过需加入粉丝团的福袋。关闭可捕获更多候选。 |
| 只抢实物 | 仅上报实物奖品福袋。普通扬声红包仍会在服务端被筛选。 |
| 优先大红包 | 默认开启，越大的人均红包越优先被处理，阈值默认 40（可自己调整）。 |
| 启用筛选服务 | 使用 Go 服务统一筛选与调度，连接异常时可关闭以回退本地逻辑。 |
| 抢后停留 | 参与结束后在直播间停留的秒数，期间可以做补扫或互动。 |
| 自动抢红包 | 开启后在直播间中尝试红包，关闭只参与福袋。 |

## 四、常见场景与建议
- **扫不到福袋**：先确认服务端地址可访问、过滤不要全部打开、日志中 `444` 错误较少。
- **想提高中奖率**：添加多个账号分工，开启优先大红包与抢后停留，告诉服务端保留扫描名额。
- **服务端候选池空**：查看 `/pool/top?limit=30`，如果空就等候端口恢复或多账号扫描。
- **账号登录失败**：关闭对应账号的“启动”按钮，删除重来或者清除 `tokens.json` 后重新扫码。

## 五、输出与统计
- 顶部统计卡片实时刷新今日收益（抖币）、成功率与服务端租约状态。
- 账号列表显示每个账号的运行状态、实时总数、中奖钻石、消耗、参与次数。
- 福袋面板、历史记录分别展示正在参与与最近完成的福袋，成功或失败都会记录。
- 日志面板可以切换“日志 / 学习”，学习模式展示服务端复评分、学习桶信息。

## 六、打包与部署（可选）
- 需制作可分发安装包时运行 `npm run electron:build` 或 `npm run electron:build:win`。
- 若使用 Go 服务共享池，可执行 `npm run pool:server:go` 本地运行，或使用 `npm run pool:docker:up` 快速部署。

## 七、故障排查小贴士
- `Electron failed to install`：删除 `node_modules/electron` 重新 `npm install`。
- 搜不到候选：降低同时活跃账号数、缩短分类扫描页数、开启日志中的“扫描池”检查详情。
- 风控报 `Non-JSON, 444`：加长详情请求间隔、减少并发、等待 45～70s 冷却。

## 八、注意事项
- 保持 `tokens.json` 在 `.gitignore` 之外的目录，切勿上传到仓库。
- Go 服务仅处理候选，不保存任何登录态；账号登录仍在本地。
- 如果服务端不可用可临时关闭“启用筛选服务”退回本地调度。
